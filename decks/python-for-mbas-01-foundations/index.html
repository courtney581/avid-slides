What is Python?
What it is
A general-purpose programming language
We write instructions in plain text, the computer runs them. No compiling, no setup magic.
Why it matters

Why do you care?
Python doesn't make you a programmer. It makes you fluent in what your team is doing — and unlocks what's possible.

Indentation matters
Python is whitespace-sensitive — indentation defines which lines belong inside if, for, or def. No curly braces. 4 spaces per level is the standard.
Colab auto-indents — let it. If you get an IndentationError, count your spaces.
GOOD ✓
if cups > 100:
    print("Busy day")
BAD ✗
if cups > 100:
print("Busy day")  # IndentationError

Functions

A name for some code — a first taste
def total_for(qty, price):
    return qty * price

print(total_for(3, 4.50))   # → 13.5
print(total_for(1, 6.95))   # → 6.95

How do I name a chunk of logic and reuse it?

1
def NAME(args): defines a function
2
Indented body holds the logic
3
return sends the result back
4
Call with NAME(values) — Python plugs them in
Why functions matter
You already use functions all the time — print(…), len(…), type(…).
Writing your own means you can name your logic and reuse it everywhere — instead of pasting the same code into 8 cells.
Why write your own?
Reuse — write once, call many times
Read — total_for(3, 4.50) says what it does
Refactor — fix it once, every caller benefits
Claude-friendly — easier to ask for help

Section 06

Working with
Claude

Your coding partner — not your replacement
How to prompt Claude when you're stuck
When something doesn't work, Claude can help — but how you ask matters.
📋
Show your code
Paste the cell that's misbehaving in full, not just a summary
⚠️
Show the error
Paste the entire traceback, including the last line — that's the actual error
🎯
Say what you expected
"I expected 5 cups but it printed 0" — gives Claude the goal, not just the symptom
The clearer your question, the better Claude's answer.
A good Claude prompt
You
"I'm trying to count how many transactions are over £5. Here's my code: [paste 5 lines]. I expected 3 but got 0. What am I doing wrong?"
Claude
"Your loop is comparing tx (a dict) to a number. You meant tx['price']. Here's the fix: [3 corrected lines]. Try this — it should give you 3."
Notice — code, expectation, and the specific question. Claude responded with a targeted fix, not a guess.
The two-try rule
"Try it twice yourself before you ask Claude."
FIRST TRY
Write the code as best you can
SECOND TRY
Read the error, change one thing
THEN
Paste it into Claude with context
Reaching for Claude before thinking means you never learn the syntax. The rule keeps Claude as a partner, not a crutch.
