Section 02

Why
functions?

The DRY principle
When you find yourself pasting the same code with one small change — it's time for a function.
BEFORE
print(f"Astoria sold 312 cups")
print(f"Hell's K sold 287 cups")
print(f"Lower M. sold 195 cups")
AFTER
def report(store, n):
    print(f"{store} sold {n} cups")

report("Astoria", 312)
report("Hell's K", 287)
report("Lower M.", 195)
DRY — Don't Repeat Yourself. Same output, far less code, one place to fix bugs.
Defining a function
def
The keyword that says "I'm defining a function"
name
What you'll call it later (your choice)
(parameters)
Inputs the function expects
: & indent
The body of the function
return
The value sent back to the caller
def revenue(qty, price):
    total = qty * price
    return total

morning = revenue(3, 4.50)   # → 13.5
evening = revenue(8, 5.95)   # → 47.6

How do I take inputs and give back an answer?

1
(qty, price) are parameters — placeholders for inputs
2
return total sends the result back
3
Call with values — Python plugs them in
4
Each call is independent
def report(store, product, cups):
    return f"{store} sold {cups} {product}s"

# Positional — order matters
report("Astoria", "Latte", 312)

# Named — order doesn't matter
report(cups=312, product="Latte",
       store="Astoria")

Why give arguments names when calling?

1
Positional is fine for 2-3 args
2
Named (keyword) arguments are clearer when the function has many params
3
You can mix — positional first, then named
4
Named arguments make your code self-documenting
def revenue(qty, price, discount=0):
    total = qty * price * (1 - discount)
    return total

revenue(10, 4.50)              # → 45.0
revenue(10, 4.50, discount=0.1) # → 40.5

How do I have an optional input?

1
discount=0 is a default — used if caller doesn't pass it
2
Defaults must come after non-default params
3
Keeps the simple case simple, the flexible case possible
def revenue(qty, price, discount=0):
    """Return total revenue after a discount.

    qty:      number of items sold
    price:    unit price
    discount: 0.1 = 10% off (default 0)

    return qty * price * (1 - discount)

# Before — explicit loop
prices = [3.00, 4.50, 6.95, 2.50]
in_pence = []
for p in prices:
    in_pence.append(p * 100)

# After — list comprehension
in_pence = [p * 100 for p in prices]

How do I express a loop in one line?

1
Read it left-to-right: "for each p in prices, compute p × 100"
2
Result is a new list
3
Once you see this, you'll see it everywhere — pandas, NumPy, GenAI-generated code
prices = [3.00, 4.50, 6.95, 2.50, 5.10]

expensive = [p for p in prices if p > 4]
# → [4.50, 6.95, 5.10]

How do I filter in a list comprehension?

1
Add if CONDITION at the end
2
Only items where the condition is true survive
3
Same logic as for + if + append — just compressed
prices = [3.00, 4.50, 6.95, 2.50, 5.10]

# Halve every price below £4 — return a new list
cheap_half = [[?] for p in prices if [?]]

print(cheap_half)

Batteries included — and you only have to know they exist
What's a library?
A library (or "module") is pre-written Python that someone else has packaged up. You use import to bring it in, then call its functions. Python ships with dozens of standard libraries — already installed, no setup. 
import math
import random
import datetime

These four are today's focus

What's the syntax for using a library?

Section 05

Reading
errors

What Python is actually telling you — and how to ask Claude
The shape of a traceback
Traceback (most recent call last):
  File "<cell line 3>", line 3, in <module>
    total = qty + price
TypeError: unsupported operand type(s)
for +: 'int' and 'str'
The bottom-up rule
Read the last line first — that's the actual error
Then the line before — where it happened
Then the type — TypeError, NameError, KeyError tell you what kind of mistake
Tracebacks look scary. They aren't — they always tell you exactly where and why.
The errors you'll meet most
ErrorWhat it meansLikely fix
NameErrorPython doesn't know this nameYou misspelled a variable, or forgot to define it
TypeErrorWrong type for the operationProbably mixing str and int — try int() or float()
KeyErrorDict doesn't have that keyCheck the spelling and capitalisation of the key
IndentationErrorWrong whitespaceCount your spaces — 4 per level
SyntaxErrorThe code isn't valid PythonUsually a missing : ( or quote
qty = 5
price = 4.50

total = qnty * price
# NameError: name 'qnty' is not defined

# Fix: qty, not qnty
total = qty * price    # → 22.5

What does Python actually tell me?

1
NameError: name 'qnty' is not defined — the name doesn't exist
2
Look — qnty is missing a "u"
3
Fix the typo, run again
Using Claude as your debugger
You
"I got this error: TypeError: unsupported operand type(s) for +: 'int' and 'str'. Here's my code: [paste 4 lines]. What's wrong?"
Claude
"On line 3 you're adding qty (an integer) to price, which is a string because you read it from a CSV. Wrap price in float(): total = qty + float(price)."
Paste the code, paste the whole error, and the fix is usually one message away.
The two-try rule
"Try it twice yourself before you ask Claude."
FIRST TRY
Write the code, run it, see what Python says
SECOND TRY
Read the traceback bottom-up; change one thing
THEN
Paste the code + error + expectation into Claude
The error message is Claude-readable. So is the question "why did this break?" Asking blind isn't.
