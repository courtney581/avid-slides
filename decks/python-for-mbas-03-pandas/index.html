Meet
pandas

What is pandas?
What it is
The Python library for tabular data
Think Excel sheet, think SQL table — same idea, but you can run any analysis on millions of rows in seconds.
Why it matters
It's the industry standard
Every data team, every analyst, every Kaggle notebook. Once you can read pandas, you can read 80% of the data code you'll ever see.
A DataFrame is just a table
A DataFrame is a table — rows and columns, with a name on each column. Each column has a type. The index is a row label — usually just 0, 1, 2…

Section 03

Selecting &
filtering

df["unit_price"]            # the whole column
df["unit_price"].head()     # first 5
df["unit_price"].mean()     # average → 3.38

How do I grab one column?

1
df["col_name"] — square brackets, name in quotes
2
A single column is a Series — like a list with an index
3
Series have methods — .mean(), .sum(), .max(), .value_counts()
df[["product_category",
    "unit_price",
    "transaction_qty"]].head()

How do I select a subset of columns?

1
Pass a list of names inside df[…] — note the double brackets
2
Returns a smaller DataFrame
3
Order of names in the list = order in the output
.loc vs .iloc
.loc[ ] — by label
Use the actual values of the index and column names. "Give me the rows where the index is X."
.iloc[ ] — by position
Use integer positions. "Give me row number 5."
df.loc[0:4, "unit_price"]     # rows 0..4 (inclusive) by label
df.iloc[0:5, 3]              # rows 0..4 by position, column 3
90% of the time you'll use boolean filters instead of either — next slide.
# All transactions in Hell's Kitchen
hk = df[df["store_location"] == "Hell's Kitchen"]

# All transactions over $10
big = df[df["transaction_qty"]
         * df["unit_price"] > 10]

hk.shape    # (49894, 10)
big.shape   # (12831, 10)

How do I keep only the rows I care about?

1
Inside df[…] write a condition — pandas keeps rows where it's True
2
Reads exactly like SQL's WHERE
3
The condition is itself a Series of True/False — useful to know when things break
# Hell's Kitchen AND lattes
q = df[
    (df["store_location"] == "Hell's Kitchen")
    & (df["product_type"] == "Latte")
]

# Coffee OR Tea
q = df[
    (df["product_category"] == "Coffee")
    | (df["product_category"] == "Tea")
]

How do I combine filters?

1
Use & for AND and | for OR (not and / or — pandas is different)
2
Every condition in its own parentheses — the #1 pandas error
3
~ means NOT — ~(df["x"] == 5)
# Find: all coffee transactions in Astoria
# where the quantity is at least 2

result = df[
    [?]
]

print(result.shape)
print(result.head())

Section 04

Reshaping &
deriving

Adding columns, vectorised maths, sorting
df["revenue"] = (df["transaction_qty"]
                 * df["unit_price"])

df[["product_type", "transaction_qty",
    "unit_price", "revenue"]].head()

How do I add a derived column?

1
df["new_col"] = expression — that's it
2
Pandas runs the expression on every row at once — more on this next slide
3
The new column lives on the DataFrame from then on
Vectorisation — the "aha" moment
If you've used Excel, you've already used vectorisation.
THE WRONG WAY ✗
revenue = []
for i in range(len(df)):
    r = (df["transaction_qty"][i]
         * df["unit_price"][i])
    revenue.append(r)
df["revenue"] = revenue
# ~12 seconds on 149k rows. Don't.
THE RIGHT WAY ✓
df["revenue"] = (df["transaction_qty"]
                 * df["unit_price"])

# ~0.05 seconds. 240× faster.
Pandas does the loop in C, not Python. Multiply two columns directly — never loop unless you have to.
# Sort one column, descending
df.sort_values("revenue", ascending=False).head(10)

# Sort by two columns
df.sort_values(
    ["store_location", "revenue"],
    ascending=[True, False],
).head(10)

How do I sort the table?

1
.sort_values(col) sorts ascending by default
2
ascending=False for descending
3
Pass a list to sort by multiple columns
4
Default sort is non-destructive — assign back to keep it
df = df.drop(columns=["transaction_id"])

# Or pick the ones to keep
df = df[["transaction_date", "store_location",
         "product_category", "revenue"]]

How do I remove a column?

1
.drop(columns=[…]) removes by name
2
Sometimes simpler to list the ones you want and reassign
3
Either way — the column you removed is gone from df
df["transaction_date"] = pd.to_datetime(
    df["transaction_date"])

df["hour"] = pd.to_datetime(
    df["transaction_time"],
    format="%H:%M:%S").dt.hour

df["weekday"] = df["transaction_date"].dt.day_name()
df["month"]   = df["transaction_date"].dt.month_name()

How do I work with dates?

1
pd.to_datetime(col) converts strings to real datetimes
2
Once converted, .dt.X gives you year/month/day/hour/weekday for free
3
Always parse dates first — most date bugs come from forgetting this
df.isna().sum()
# unit_price         0
# promo_code    142891   ← lots missing

# Drop rows missing a required column
df.dropna(subset=["unit_price", "transaction_qty"])

# Or fill with a default
df["promo_code"] = df["promo_code"].fillna("none")

What do I do about empty cells?

1
.isna() returns True where missing — .sum() totals them
2
.dropna(subset=[…]) drops rows missing in those columns
3
.fillna(x) fills with a default
4
Decide per column — sometimes empty means something
Section 05

GroupBy &
aggregate

The pandas finale — your SQL GROUP BY muscle memory
df.groupby("store_location")["revenue"].sum()
# Astoria            199824.45
# Hell's Kitchen     236511.17
# Lower Manhattan    230057.34

What's the daily takings by store?

1
df.groupby("col") segments rows by unique values
2
Pick the column to aggregate, then call .sum() / .mean() / .count()
3
Result is a Series indexed by group
4
Same idea as SQL's GROUP BY
df.groupby("store_location")["revenue"] \
  .agg(["sum", "mean", "count"])

df.groupby("product_category").agg(
    total_revenue=("revenue", "sum"),
    avg_price=("unit_price", "mean"),
    n_transactions=("transaction_id", "count"),
)

How do I get sum + average + count in one go?

1
.agg([…]) runs multiple aggregations at once
2
Named-tuple form lets you name the output columns
3
Result is a DataFrame — chain .sort_values() straight after
# Top 5 product types by revenue
top5 = (
    df.groupby("product_type")["revenue"]
      .sum()
      .sort_values(ascending=False)
      .head(5)
)
print(top5)

What are the top 5 products by revenue?

1
Chain methods on multiple lines — wrap in parentheses
2
groupby → sum → sort → head — read top to bottom
3
This is the canonical pandas idiom — you'll write it constantly
# Hour of day with highest revenue
# at Hell's Kitchen — use the `hour` column

hk = df[[?]]
by_hour = (hk.groupby([?])[[?]]
             .sum()
             .sort_values([?]))

print(by_hour.head(3))

groupby hour, aggregate revenue with .sum()

# Average daily revenue by day of the week

daily = (df.groupby([[?], [?]])[[?]]
            .sum()
            .reset_index())

by_weekday = (daily.groupby([?])[[?]]
                   .mean()
                   .sort_values([?]))

print(by_weekday)

Average daily revenue by weekday

1
Group by transaction_date + weekday for a per-day total
2
Then group those totals by weekday and take the mean
3
Sort descending — weekends busier than midweek

Errors &
try / except

Pandas errors you'll meet
ErrorWhat it meansLikely fix
KeyError: 'cust_id'Column doesn't existCheck spelling — df.columns shows them all
TypeError after .dt.hourColumn isn't a datetimeYou forgot pd.to_datetime(...) first
SettingWithCopyWarningPandas isn't sure what you meanSee next slide — usually safe to ignore
ValueError: cannot convertType mismatch in a filterMake sure both sides match (str vs int)
The most-Googled warning in data
SettingWithCopyWarning:
A value is trying to be set on a copy of a slice from a DataFrame.
80% of the time it's harmless — your code worked
20% of the time you assigned to a copy of the data, not the original
The safe fix: use .loc[...] instead of chained brackets, or call .copy() after a filter
If it's blocking your work, paste it into Claude with your code. It's a warning, not an error.
for value in df["unit_price"]:
    try:
        n = float(value)
    except ValueError:
        print(f"Skipping bad value: {value}")

What if some values are unparseable?

1
try: runs the risky code
2
except ERROR_TYPE: catches a specific failure
3
Always catch a specific exception — never a bare except:
4
Most pandas work doesn't need this — but it's there when you do

Using Claude
with pandas

Pandas has 600+ methods. Claude knows them all. Here's how to ask.
Two ways Claude helps with pandas
🔎

Method finder

"I have a DataFrame and I want to do X. What's the pandas method?"
🐛

Debugger

"I got this error / unexpected output. Here's my code and what I expected. What's wrong?"
Pandas is too big to memorise. The skill is knowing what to ask — and recognising the right answer.
Showing Claude your DataFrame
You
"Here's the first 3 rows of my DataFrame as a dict: [paste df.head(3).to_dict()]. How do I find the average revenue per product category?"
Claude
"Use df.groupby('product_category')['revenue'].mean().sort_values(ascending=False). This groups by category, averages revenue, and sorts highest first."
Paste the shape of your data, not all 150k rows. .head(3).to_dict() gives Claude exactly enough.
Ask better questions
When Claude gives you pandas, don't stop there — the follow-up is where you actually learn.
"Explain each line"
Ask Claude to comment every method call
"Why is this the right shape?"
Ask whether the result is a Series or DataFrame, and why
"Is there a more idiomatic way?"
Pandas often has 3 ways — ask for the most readable
The best Python answers come from asking the best pandas questions.
