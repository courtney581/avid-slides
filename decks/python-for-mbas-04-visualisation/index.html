Section 02

Why
visualise?

Why visualise?
The case
A chart tells a story in half a second
A table of 12 numbers is hard to read. A chart of 12 numbers is instant. Visualisation is how you make your analysis persuasive in a meeting.
Why it matters
MBAs live in decks
Knowing how to make a chart that says one clear thing — without an analyst — is leverage.
Picking the right chart
Bar chart
Compare categories — e.g. revenue by store
Line chart
Show change over time — e.g. daily revenue Jan → Jun
Histogram
See the distribution of a single number — e.g. transaction size
Scatter plot
Show the relationship between two numbers — e.g. price vs quantity
Section 03

Matplotlib
basics

The original Python plotting library. Ugly by default — but everyone uses it.
import matplotlib.pyplot as plt

by_store = df.groupby(
    "store_location")["revenue"].sum()

plt.bar(by_store.index, by_store.values)
plt.title("Revenue by store")
plt.show()

How do I make a chart?

1
import matplotlib.pyplot as plt — the universal alias
2
plt.bar(labels, values) — bar chart
3
plt.show() displays it in Colab
4
One chart per cell — keeps things simple
plt.figure(figsize=(10, 5))
plt.bar(by_store.index, by_store.values,
        color="#2899A4")
plt.title("Revenue by store", fontsize=16)
plt.xlabel("Store")
plt.ylabel("Revenue (USD)")
plt.show()

How do I make it not embarrassing?

1
figsize=(w, h) sets the size — bigger is usually better
2
color accepts hex codes — match your brand
3
.title(), .xlabel(), .ylabel() — never ship a chart without these
daily = (
    df.groupby("transaction_date")["revenue"]
      .sum()
      .sort_index()
)

plt.figure(figsize=(12, 5))
plt.plot(daily.index, daily.values,
         color="#2899A4")
plt.title("Daily revenue · Jan–Jun 2023")
plt.show()

Is the business growing?

1
Use .sort_index() so dates plot in order
2
plt.plot(x, y) — same shape as plt.bar
3
Already useful — you can spot weekly cycles and a clear trend
plt.figure(figsize=(10, 5))
plt.hist(df["revenue"], bins=40,
         color="#E05FA0")
plt.title("Distribution of transaction sizes")
plt.xlabel("Revenue per transaction (USD)")
plt.ylabel("Count")
plt.show()

What does a 'normal' coffee transaction look like?

1
plt.hist(values, bins=N) — N is the number of buckets
2
Use histograms when you want shape, not exact values
3
Long-tail distributions are everywhere in business data

Section 04

Seaborn

Same plots, fewer keystrokes, defaults that don't embarrass you
Why seaborn
Seaborn is built on top of matplotlib. Same plot types, but with smarter defaults, prettier colours, and one-liners for things matplotlib makes you write 5 lines for.
When you can use seaborn, use seaborn. Drop back to matplotlib for fine control.
MATPLOTLIB
by_store = df.groupby("store_location") \
             ["revenue"].sum()
plt.bar(by_store.index, by_store.values,
        color="#2899A4")
SEABORN
sns.barplot(data=df, x="store_location",
            y="revenue")
import seaborn as sns
sns.set_theme(style="whitegrid")

plt.figure(figsize=(10, 5))
sns.barplot(
    data=df,
    x="store_location",
    y="revenue",
    estimator="sum",
    palette="crest",
)
plt.title("Total revenue by store")
plt.show()

What does the seaborn equivalent look like?

1
data=df — pass the whole DataFrame
2
x and y are column names (strings)
3
estimator="sum" — seaborn defaults to mean
4
Palettes — try crest, flare, mako
plt.figure(figsize=(12, 5))
sns.boxplot(
    data=df,
    x="product_category",
    y="revenue",
    palette="crest",
)
plt.title("Transaction size by product category")
plt.show()

Which categories have the biggest — and most variable — transactions?

1
Box = middle 50%; line = median; dots = outliers
2
Tells you far more than just the average
3
Common in finance and ops dashboards
pivot = df.pivot_table(
    index="weekday",
    columns="hour",
    values="revenue",
    aggfunc="sum",
)

plt.figure(figsize=(14, 6))
sns.heatmap(pivot, cmap="crest",
            linewidths=0.5)
plt.title("Revenue heatmap — weekday × hour")
plt.show()

When are people actually buying coffee?

1
pivot_table reshapes the data — weekday on rows, hour on columns
2
sns.heatmap(pivot) colours by value
3
The morning rush jumps out instantly — that's the power of viz

Section 05

Plotly —
interactive charts

The 'wow' moment
import plotly.express as px

daily = (
    df.groupby(["transaction_date",
                "store_location"])["revenue"]
      .sum()
      .reset_index()
)

fig = px.line(
    daily, x="transaction_date", y="revenue",
    color="store_location",
    title="Daily revenue by store",
)
fig.show()

What if I want the user to hover and zoom?

1
plotly.express — same family as seaborn (one-liners over a DataFrame)
2
color= automatically separates lines per category
3
Hover, zoom, toggle legend — all free
4
Great for dashboards and stakeholder share

import requests

r = requests.get(
    "https://api.exchangerate.host/latest?base=GBP")

print(f"1 GBP = {rates['USD']} USD")
print(f"1 GBP = {rates['EUR']} EUR")

print(msg.content[0].text)
