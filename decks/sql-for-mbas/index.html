What is SQL?
Stands for
Structured
Query
Language
A language we use to query and ask for information from large datasets.
Why it matters
SQL is ubiquitous
Almost every large business employs analysts to answer business questions using SQL. It's the universal language of data.
Why do you care?
SQL enables you to interrogate how a business is actually performing. A basic understanding makes you:
Better informed
Make data-driven decisions with confidence, not guesswork.
Less reliant on analysts
Self-serve answers to common data questions without waiting in the queue.
More confident
Understand the data behind the dashboards your team presents.
More 'dangerous' in meetings
Know when the numbers don't add up — and be able to ask the right questions.

Why Excel doesn't scale
Excel spreadsheets lead to redundancy. Relational databases eliminate it — storing each entity once and linking tables by keys.
The Excel way
One flat table, repeated data
order_id customer_name customer_city product_name product_category product_price
1001 John Smith London Wireless Mouse Electronics 25
1002 John Smith London Keyboard Electronics 45
1003 John Smith London Laptop Stand Accessories 30
1004 Sarah Jones Manchester Monitor Electronics 200
1005 Sarah Jones Manchester Keyboard Electronics 45
1006 Ali Khan Birmingham Monitor Electronics 200
1007 Ali Khan Birmingham Wireless Mouse Electronics 25
1008 Emma Brown Leeds Laptop Stand Accessories 30
Customer details are repeated on every order row — storage balloons and updates are error-prone.
The SQL way
Related tables, stored once
Orders
order_id customer_id product_id
1001 C1 P1
1002 C1 P2
1003 C1 P3
1004 C2 P4
1005 C2 P2
Customers
customer_id customer_name customer_city
C1 John Smith London
C2 Sarah Jones Manchester
C3 Ali Khan Birmingham
C4 Emma Brown Leeds
Products
product_id product_name product_category product_price
P1 Wireless Mouse Electronics 25
P2 Keyboard Electronics 45
P3 Laptop Stand Accessories 30
P4 Monitor Electronics 200
primary key foreign key
Each entity lives in one table. Primary keys identify rows; foreign keys link tables — cheaper, cleaner, scales to billions of rows.

Summary
SELECT / FROM
Every query needs these two. Use * for all columns, or name individual columns.
WHERE
Filters rows. Use comparison operators: =, !=, >, BETWEEN, ILIKE
ORDER BY
Sorts results. Add DESC for descending, ASC for ascending.
COUNT / DISTINCT
Aggregators enable summary statistics. DISTINCT returns unique entries.

When querying a dataset we're often interested in summary statistics rather than individual rows. This is where aggregators come in.

3

Aliasing
In SQL it is possible to alias a column (or a table) — simply giving it a more descriptive name which can then be used throughout the rest of the query.
BASIC ALIAS
SELECT weighted_salary_usd
       AS salary
FROM business.business_schools;
WITH AGGREGATION
SELECT  primary_location AS country
      , AVG(weighted_salary_usd) AS avg_sal
FROM business.business_schools
GROUP BY country
ORDER BY avg_sal DESC;
Why alias?
Makes output easier to read
Required when naming computed columns
Can reuse alias in ORDER BY
Shortens long expression names
SELECT   [col]
       , [aggregate] AS [new_name]
FROM business.business_schools
GROUP BY [col]
ORDER BY [new_name] DESC;

CASE WHEN
The business.business_schools dataset doesn't natively include a "US vs International" field — but it does include primary_location.
Using CASE WHEN logic allows us to manipulate the data beyond what natively exists — like creating new derived columns on the fly.
Think of it as IF/ELSE
CASE WHEN condition
THEN result
ELSE fallback
END AS column_name
SELECT  school_name
      , primary_location
      , CASE WHEN
          primary_location != 'US'
          THEN 'International'
          ELSE primary_location
          END AS region
FROM business.business_schools;

How many schools are in the US versus everywhere else in the world?

1
CASE WHEN checks a condition row by row
2
THEN sets the value if the condition is true
3
ELSE sets the fallback value
4
END AS region closes the CASE and names the new column
SELECT
  CASE WHEN primary_location != 'US'
       THEN 'International'
       ELSE primary_location
       END AS region
  , COUNT(*) AS num_schools
FROM business.business_schools
GROUP BY region
ORDER BY num_schools DESC;

How many schools are in the US versus everywhere else?

1
You can GROUP BY the aliased CASE WHEN column
2
This creates a US / International split on the fly — no new data needed
Expected output
US · 43 schools
International · 57 schools

  CASE WHEN weighted_salary_usd > 200000

How many schools fall into each pay category?

Create a column that labels schools as:

SELECT

What about GenAI?
Always been normal
Even before GenAI, analysts always used Google & StackOverflow. Literally no one knows all commands by heart — and they never have.
GenAI is not infallible
It's not uncommon to get responses that just don't work at all. Models are trained to be sycophantic — if you ask something wrong, it may keep giving answers that will never work.
Know the basics first
If you can write basic SQL yourself, GenAI will make you 10× faster.

If you only use GenAI, it can make you 10× slower.
